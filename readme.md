## О структуре проекта
Проект разделён на несколько независимых модулей, каждый из которых базируется в своей директории.\
На момент этого коммита, готовы все четыре модуля: model, backend, tgbot и frontend. В model мы обучаем модель, в backend - работаем с веб-сервером, tgbot - собственно, ТГ-бот.\
В каждой директории есть свой readme-файл, который более детально описывает именно свой модуль.

## Как работать с проектом
Находясь в какой-либо папке (model, backend или какой-либо еще из модулей), мы работаем конкретно с этой частью проекта.\
Работая, скажем, над веб-сервером, нам не нужно думать о том, хорошо ли датасет подобран для модели или какая версия YOLO использовалась для обучения - этим занимается совсем другая часть проекта.\
Но тем не менее, весь проект выстроен в некий конвейер: размечаем данные в Roboflow, оттуда импортируем в папку model/dataset и обучаем модель; веса обученной модели копируем в рабочую папку веб-сервера, который использует их для обработки изображений, полученных с фронтенда.\
Такая структура подразумевает беспроблемное обновление модели: импортируем новый датасет -> ```python train.py``` -> сохраняем веса в папку *trained_models*.\
Затем сохраненный файл весов можно копировать в *backend/* и использовать на веб-сервере.

Более подробно о каждом модуле проекта можно прочитать в соответсвующему ему readme-файле.

На момент коммита, проект уже находится в рабочем состоянии, Вам нужно только импортировать размеченный датаест и установить недостающие Python-модули.

### Как запустить веб-приложение
1) Переходим в папку backend
2) Запускаем комманду ```uvicorn main:app``` - запустился сервер (стандартно, на http://127.0.0.1:8000)
3) Переходим в папку frontend
4) Запускаем ```npm run dev``` и переходим по выведенному адресу - в браузере откроется наше веб-приложение

### Как запустить тг-бота
1) Перейти в папку tgbot
2) Выполнить ```python main.py```
3) В Телеграме выходим на бота @MeasurePlantbot
