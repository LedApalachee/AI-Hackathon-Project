## Структура этой папки
Папка **segmented**:\
В эту папку веб-сервер сохраняет обработанные изображения и связанные с ними измерения, хранящиеся в текстовых файлах.\
В последствии эти изображения и текстовые файлы становятся доступны для фронтенда.\
**Важно!** Если этой папки не существует, то создайте ее - без нее веб-сервер выдаст ошибку и не сможет работать.

**main.py**:\
Главный исполняемый файл, который является точкой запуска веб-сервера: ```uvicorn main:app```\
"Монтирует" папку *segmented* для доступа извне; реализует единственный энд-пойнт: "/upload-zip".
Использует модуль измерения растений из файла *measure_plant.py*.

**measure_plant.pt**:\
Файл весов обученной модели, использующейся на сервере.\
Импортируется из *model/trained_models/*.\
! Обязательно переименовать в *measure_plant.pt* при копировании.

**measure_plant.py**:\
"Модуль измерения растений". Используется в *main.py*.\
Может запускаться без веб-сервера (см. "Использоввание коммандной строки").

**requirements.txt**:\
Список использующихся third-party модулей Python.\
Чтобы установить эти зависимости, воспользуйтесь коммандой ```pip install -r requirements.txt```\
Следует иметь ввиду, что данный список составлялся вручную, чтобы не вписать лишние модули, не имеющие отношения к делу, и поэтому некоторые зависимости могли быть пропущены, и их придется установить отдельно.

## Как работает этот бэкенд
Сервер реализует единственный энд-пойнт "/upload-zip".\
Фронтенд присылат zip-архив с пачкой изображений на обработку.\
Важно учитывать, что эти изображения должны находится прямо в архиве - не внутри какой-либо папки.\
Изображения проходят через обработку моделью и сохраняются в директорию *segmented* в уникальную папку, название которой формируется на основе uuid.\
Сервер формирует к каждому изображению в *segmented* текстовый файл с соответствующими результатами измерений.\
Возвращаемые данные - список *results* словарей, каждый из которых имеет два поля: *image_url* и *txt_url*. По этим ссылкам фронтенд получает доступ к обработанным изображениям и текстовым файлам измерений.

## Использование коммандной строки:
Инструмент по измерению площедей и длин можно использовать, не прибегая к запуску веб-сервера, а используя средства коммандной строки.\
В этом случае он будет выводить в консоль посчитанные длины/площади каждого найденного листочка/корня/стебля в каждом из входных изображений.

Общий вид комманды:
```
python measure_plant.py [arg 1: файл весов] [arg 2: папка входных изображений] [arg 3: папка для сохранения результатов] [arg 4: масштаб пиксели/мм]
```

В качестве первого аргумента можно передать точку: в этом случае скрипт будет искать в текущей директории файл весов с тем же именем что и сам скрипт, но с расширением файла .pt:
```
python measure_plant.py . images results 9  # будет искать файл весов measure_plant.pt
```

Вторым аргументом скрипт принимает относительный путь папки с входными изображениями

Третий аргумент - относительный путь папки, куда сохранятся файлы-результаты (если указанной директории не существует, она создастся сама)\
Если аргумент передан как символ точки, то файлы сохранятся в текущую директорию:
```
python measure_plant.pt mymodel.pt images . 10
```

### Примеры использования:
```
python measure_plant.py weights/mymodel.pt images results 9.2  # возьмет указанный файл весов, прогонит все картинки из папки images и сохранит результаты в results; использует масштаб 9.2 пкс/мм
```
```
python measure_plant.py . images . 10  # возьмет файл весов measure_plant.pt, прогонит картинки из images, сохранит в текущую директорию
```
