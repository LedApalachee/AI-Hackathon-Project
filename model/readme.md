## Структура этой папки

Папка **dataset**:\
Здесь хранится Ваш датасет, импортированный из Roboflow.\
В нашем случае, в этой папке было следующее содержимое: папка *train*, папка *valid*, манифест *data.yaml*.\
**Важно!** Файл *data.yaml* не всегда указывает на реальную структуру папок импортированного датасета, так что лучше проверить и исправить, если что не так (по крайней мере, нам пришлось)

Папка **trained_models**:\
В этой папке лежат файлы весов обученных моделей, которые можно использовать где-то еще - в нашем случае в веб-приложении.\
Каждый назван по шаблону *plants_dd_mm_yyyy.pt*\
Для использования обученной модели где-нибудь еще, файлы весов просто копируются в нужную Вам рабочую директорию.

Папка **yolo_weights**:\
Здесь лежат веса исходных YOLO-моделей различных версий для последующего обучения.

**train.py**:\
Скрипт обучения модели.

**requirements.txt**:\
Список использующихся third-party модулей Python.\
Чтобы установить эти зависимости, воспользуйтесь коммандой ```pip install -r requirements.txt```\
Следует иметь ввиду, что данный список составлялся вручную, чтобы не вписать лишние модули, не имеющие отношения к делу, и поэтому некоторые зависимости могли быть пропущены, и их придется установить отдельно.


## Процесс обучения модели
Модель обучается на датасете, хранящемся в одноименной директории.\
Когда Вы запускаете комманду ```python train.py```, скрипт инициирует модель, используя один из файлов весов из папки *yolo-weights*, который указан в скрипте в строчке ```model = YOLO("yolo-weights/yolo26s-seg.pt")```.\
По завершению обучения, появится директория *runs* со своими поддиректориями.\
Итоговые веса модели сохранены в *runs/segment/train/*.\
Для дальнейшего экспорта модели в другие проекты рекоммендуется скопировать файл *best.pt* в папку *trained_models* и переименовать его в соответствии с упомянутым выше шаблоном.
